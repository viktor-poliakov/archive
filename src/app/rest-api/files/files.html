Файлы и бинарные данные

До сих пор мы гоняли по сети текст — в основном JSON. Но приложению часто нужно отправить или получить файл: загрузить аватар, скачать отчёт в PDF, выгрузить архив. Это байты, а не текст, и для них есть свои инструменты: FormData — чтобы отправлять, и Blob — чтобы получать и хранить бинарные данные.

Аналогия — почтовая посылка. Обычный запрос это открытка: текст на виду. Файл — это коробка: её нужно упаковать (FormData), а на приёмке — распаковать содержимое (Blob). Разберём обе стороны.

Форматы тела: не только JSON

В теле запроса и ответа может лежать что угодно — сервер понимает, что именно, по заголовку Content-Type. На практике вам встретятся три варианта:

  • JSON-строка — структурные данные (Content-Type: application/json). Основной формат REST.
  • FormData — форма с файлами и полями (multipart/form-data). Так браузер отправляет загрузку файла.
  • Blob / ArrayBuffer — «сырые» байты (картинка, PDF, архив). Так удобно получать файлы.

FormData: отправка файлов и форм

FormData — это набор пар «ключ — значение», где значением может быть и текст, и файл. Вы наполняете его через append и передаёте прямо в body. Файл обычно берут из элемента <input type="file">:

Не ставьте Content-Type вручную! Это самая частая ошибка с FormData. Браузер сам выставит multipart/form-data и добавит boundary — уникальный разделитель между частями. Если задать заголовок руками, boundary потеряется, и сервер не сможет разобрать, где кончается одно поле и начинается следующее.

Blob: бинарные данные как объект

Blob (Binary Large Object) — это неизменяемый «контейнер» байтов с указанием типа. Когда сервер отдаёт файл, ответ читают методом res.blob() — так же, как JSON читают через res.json(). Blob можно и создать самому — например, чтобы собрать файл прямо в браузере:

Скачивание файла

В браузере нет функции «сохрани этот Blob на диск». Приём такой: создаём для Blob временный адрес через URL.createObjectURL, вешаем его на невидимую ссылку <a> с атрибутом download и «нажимаем» её кодом. Так работает и файл с сервера, и сгенерированный на клиенте:

Не забудьте revokeObjectURL. Object URL держит Blob в памяти, пока вы его явно не отпустите. Без revoke при частых скачиваниях память будет только расти — до перезагрузки страницы.

Прогресс скачивания

Прогресс отправки (upload)
