Анатомия HTTP: запрос и ответ

Когда браузер или ваше приложение обращается к серверу, они общаются на языке HTTP (HyperText Transfer Protocol). Всё общение сводится к простой схеме: клиент отправляет запрос (request), сервер возвращает ответ (response). Один запрос — один ответ, как реплика и отклик в диалоге.

Аналогия — заказ по почте. Запрос похож на бланк заказа: сверху вы пишете, что именно хотите (метод и адрес товара), затем отмечаете пожелания доставки (заголовки), а в конце при необходимости вкладываете само содержимое посылки (тело). Ответ — это то, что приходит обратно: сверху штамп «выполнено» или «не найдено» (код состояния), пометки об упаковке (заголовки) и сам товар (тело). Тут нет никакой магии: и запрос, и ответ — это обычный текст со строго заданной структурой, который можно прочитать глазами.

HTTP — язык клиента и сервера

HTTP работает по модели «запрос — ответ». Инициатором всегда выступает клиент: сервер сам по себе ничего не присылает, он лишь отвечает на то, о чём его спросили. Клиент формулирует «вопрос» («дай мне пользователя с номером 42», «создай новый заказ»), а сервер даёт «ответ» — с данными или с сообщением об ошибке.

Обмен происходит асинхронно: ответ приходит не мгновенно, и всё это время программа не должна «замирать». Именно поэтому в JavaScript сетевые запросы устроены на промисах и async/await — почему так, разобрано в разделе «Событийный цикл».

Важно: и запрос, и ответ имеют одинаковую структуру из трёх частей — стартовая строка, заголовки и тело. Разобравшись с одним, вы автоматически понимаете и второе.

Из чего состоит запрос

HTTP-запрос — это текст из четырёх последовательных частей:

  • Стартовая строка — метод, путь и версия протокола. Например, GET /users/42 HTTP/1.1: «получить (метод) ресурс по адресу /users/42 (путь) по версии HTTP/1.1».
  • Заголовки — пары Имя: значение, по одной на строку. Это метаданные запроса: на какой хост идём, какой формат ждём в ответ, чем удостоверяем личность.
  • Пустая строка — обязательный разделитель, который отделяет заголовки от тела.
  • Тело (body) — сами передаваемые данные, обычно JSON. У запросов вроде GET тела нет, у POST/PUT — есть.

URL и эндпоинты

Стартовая строка ссылается на путь, а полный адрес ресурса задаёт URL. URL — это не одна сплошная строка, а несколько частей с чётким назначением:

  • Схема (https) — по какому протоколу обращаться.
  • Хост (api.example.com) — на какой сервер идёт запрос.
  • Путь (/users/42) — адрес конкретного ресурса на сервере.
  • Query-строка (?fields=name,email) — необязательные параметры после ?, разделённые &.
  • Fragment (#top) — якорь внутри страницы; на сервер он не отправляется и в API обычно не используется.

Эндпоинт (endpoint) — это адрес ресурса, к которому обращается API. В REST принято различать коллекцию (много элементов) и элемент коллекции. А query-параметры служат для фильтрации, сортировки и пагинации (выдача по страницам) — они уточняют запрос, не меняя сам путь к ресурсу:

Правило: путь описывает какой ресурс нужен (/users/42), а query-параметры — как его выдать (какие поля, какая страница). Не путайте: /users/42 и /users?id=42 — это разные вещи; первое адресует один элемент, второе фильтрует коллекцию.

Из чего состоит ответ

Ответ устроен зеркально запросу — те же три части, только вместо стартовой строки здесь статусная строка:

  • Статусная строка — версия, код состояния и текст. Например, HTTP/1.1 200 OK: версия протокола, числовой код 200 и человекочитаемая пометка OK.
  • Заголовки — метаданные ответа: тип и длина тела, правила кэширования и прочее.
  • Тело — сами данные, чаще всего JSON. Отделено от заголовков всё той же пустой строкой.

Числовой код в статусной строке — самое важное в ответе: по нему клиент понимает, успешен ли запрос. 200 — успех, 404 — ресурс не найден, 500 — ошибка на сервере. Полный разбор — на странице «Коды состояния».

Ключевой момент: тело приходит отдельно от заголовков, и его нужно прочитать — метод response.json() разбирает JSON-тело в объект. Заметьте, что fetch считает успехом любой полученный ответ, даже с кодом 404 или 500: сама передача ведь состоялась.
