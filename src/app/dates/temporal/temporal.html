Temporal API

Temporal — новый встроенный инструмент для работы с датами и временем, задуманный как замена старому Date. Он чинит почти всё, за что Date ругают: неизменяемые объекты, месяцы с единицы, нормальные часовые пояса и календари, надёжный разбор строк.

⚠️ Поддержка браузерами. Temporal — молодой стандарт. На середину 2026 года он поддержан ещё не во всех браузерах: раньше всех его включил Firefox, в Safari и Chrome он появляется постепенно (или доступен за флагом). На MDN он помечен как Limited availability (не Baseline). Для реальных проектов подключайте полифилл (@js-temporal/polyfill), а актуальный статус смотрите на MDN/caniuse. Примеры ниже показывают сам API — в браузере он может быть доступен ещё не везде.

Зачем он нужен

Temporal с самого начала спроектирован без «врождённых» болей Date:

  • Неизменяемость — методы возвращают новый объект, оригинал цел (никаких случайных правок).
  • Месяцы с 1 — январь это 1, а не 0.
  • Настоящие таймзоны и календари — встроенно, а не «UTC либо местное».
  • Разделение понятий — «просто дата», «просто время», «момент с зоной» — это разные типы.
  • Надёжный разбор строк ISO без сюрпризов.

Date и Temporal — в чём разница

  Date Temporal
Изменяемость меняется на месте неизменяемый
Месяцы 0–11 1–12
Таймзоны только UTC/местная любые, встроенно
Типы один на всё отдельные (дата/время/зона…)
Поддержка везде пока не везде (нужен полифилл)

Итог: Temporal — то, каким должен был быть Date. Пока он «докатывается» до браузеров, для нового кода берите полифилл; а базу — про старый Date — смотрите в разделе «Даты».
