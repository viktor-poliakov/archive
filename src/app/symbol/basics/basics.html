Symbol — основы

Symbol (символ) — это один из семи примитивных типов в JavaScript, наряду со строкой, числом и булевым значением. У него одна суперспособность: каждое созданное значение гарантированно уникально — другого такого во всей программе нет и быть не может.

Аналогия — заводской серийный номер. На двух одинаковых с виду телефонах наклейка с одним и тем же названием модели, но серийные номера у них разные — по номеру всегда можно отличить один аппарат от другого. Символ работает так же: описание можно повторять сколько угодно, но сам символ каждый раз новый и ни с чем не совпадает.

Как создать символ

Символ создаётся вызовом функции Symbol(). В скобки можно передать строку-описание (description): она ни на что не влияет и нужна только для удобства отладки — чтобы в консоли было видно, что это за символ.

Обратите внимание: символ создают обычным вызовом функции — Symbol(); через new нельзя — new Symbol() бросает ошибку. Тут легко запутаться: Number и String тоже можно звать без new (например, Number('1') вернёт число 1). Разница в том, что им new разрешён и создаёт объект-обёртку, а Symbol с new не работает вовсе.

Главное свойство — уникальность

Сколько бы раз вы ни вызвали Symbol() — хоть с одинаковым описанием — вы каждый раз получаете новый, ни на что не похожий символ. Два символа никогда не равны друг другу; символ равен только самому себе.

Описание: description и toString()

Прочитать описание можно через свойство .description. А метод .toString() даёт строковое представление вида Symbol(описание) — им пользуются, когда символ нужно показать человеку (например, в логах).

Symbol — это примитив

Символ — примитивное значение, а не объект, поэтому создать его через new нельзя (см. Типы). Этим он отличается от объектов-обёрток Number и String, где new формально работает и возвращает объект. Попытка же вызвать new Symbol() завершается ошибкой:

Зачем вообще нужны символы

Уникальное значение, которое невозможно случайно повторить, полезно в двух ситуациях:

  • Ключи свойств без коллизий. Символом можно пометить свойство объекта так, что оно точно не столкнётся с чужим свойством с таким же именем — см. Символы как ключи.
  • Системные «хуки». Встроенные символы (например, Symbol.iterator) — это точки, за которые сам движок JavaScript «цепляется», чтобы понять, как объект должен себя вести — см. Системные символы.
