TypeScript

TypeScript — это надстройка над JavaScript, которая добавляет в язык статическую типизацию. Вы пишете почти обычный JavaScript, но описываете, что в каких переменных и функциях лежит, — а компилятор заранее находит ошибки, которые иначе всплыли бы только во время работы программы. В браузере TypeScript не выполняется: перед запуском его превращают в обычный JavaScript.

Немного истории

Язык создала компания Microsoft, а ведущим архитектором стал Андерс Хейлсберг (Anders Hejlsberg) — тот самый человек, что стоит за Turbo Pascal, Delphi и языком C#. Первый публичный релиз состоялся в октябре 2012 года. Причина появления простая: JavaScript отлично подходит для небольших скриптов, но большие приложения на нём тяжело поддерживать — легко опечататься, передать не тот аргумент или обратиться к несуществующему свойству, и узнаёшь об этом только в рантайме.

Важный принцип, заложенный с самого начала: любой корректный JavaScript — это уже корректный TypeScript. Поэтому переходить на язык можно постепенно, файл за файлом, не переписывая проект целиком.

Ключевые вехи развития:

  • 2012 — первый публичный релиз (версия 0.8);
  • 2014 — 1.0: язык стабилизировался, исходный код открыт на GitHub;
  • 2016 — 2.0: строгая проверка на null и undefined (strictNullChecks) — одно из самых полезных нововведений. В том же году вышел Angular 2, выбравший TypeScript основным языком, что резко подняло его популярность;
  • 2018 — 3.0 и 2020 — 4.0: развитие системы типов (кортежи, условные типы, шаблонные литеральные типы);
  • 2023 — 5.0: поддержка декораторов по новому стандарту и дальнейшие улучшения. Новые версии выходят каждые несколько месяцев.

Сегодня TypeScript — один из самых востребованных языков в вебе. На нём написаны Angular, VS Code и множество популярных библиотек, а React, Vue и Node.js полностью его поддерживают.

Что такое TypeScript

Идея типизации проста: рядом со значением вы указываете его тип — что это, число, строка, объект определённой формы и так далее. Компилятор держит эти обещания в уме и ругается, как только вы их нарушаете, — ещё до запуска кода.

Аналогия. Обычный JavaScript — как отправить письмо и ждать ответа «адрес не найден» уже после отправки. TypeScript — как строгий, но полезный редактор, который читает письмо у вас за плечом и подчёркивает ошибку прямо во время написания.

Типы стираются при компиляции

Ключевой момент: типы существуют только на этапе разработки. Компилятор проверяет их, а затем стирает — в готовом JavaScript никаких аннотаций не остаётся, и на скорость выполнения они не влияют.

Аналогия. Типы — как строительные леса вокруг дома: они нужны, пока идёт стройка, но к моменту заселения их снимают. В рантайме остаётся только «дом» — чистый JavaScript.

Зачем он нужен

Что даёт статическая типизация на практике:

  • Ошибки ловятся раньше — многие баги видны прямо в редакторе, а не после того, как их найдёт пользователь;
  • Умные подсказки — редактор знает форму ваших данных и точно подсказывает доступные свойства и методы (автодополнение);
  • Код документирует сам себя — по типам сразу понятно, что функция принимает и что возвращает, без чтения её тела;
  • Безопасный рефакторинг — переименовали поле или изменили сигнатуру — компилятор перечислит все места, которые нужно поправить;
  • Масштаб — чем больше проект и команда, тем сильнее эти плюсы: типы становятся общим договором между частями кода.

При этом аннотации нужны не всюду: во многих случаях TypeScript сам выводит типы из значений, и писать их вручную не приходится.

Установка и запуск

TypeScript ставится через npm как обычный пакет. Главный инструмент — компилятор tsc (TypeScript Compiler). Для быстрого запуска .ts-файлов в разработке удобны tsx или ts-node, которые компилируют и выполняют код одной командой.

Попробовать язык прямо в браузере, без установки, можно на официальной площадке TypeScript Playground — там же видно, в какой JavaScript превращается ваш код.

Компилятор tsc

Компилятор делает две вещи за один проход:

  • Проверяет типы — сверяет ваши аннотации и сообщает об ошибках. Если что-то не сходится, вы узнаёте об этом здесь;
  • Транспилирует код — переводит TypeScript в JavaScript нужной версии (и заодно превращает современный синтаксис в более старый, если это указано в настройках).

Проверка типов и транспиляция — независимые задачи. Поэтому в реальных сборках проверку часто оставляют за tsc, а само превращение в JavaScript поручают более быстрым инструментам (esbuild, SWC, Babel), которые типы просто игнорируют.

Файл tsconfig.json

Поведением компилятора управляет файл tsconfig.json в корне проекта: в какую версию JavaScript компилировать, какая модульная система на выходе, насколько строгими быть проверкам и куда складывать результат.

Самая важная опция — strict. Она включает целый набор строгих проверок (в том числе strictNullChecks) и, по сути, и есть тот самый TypeScript, ради которого его берут. В новых проектах её стоит держать включённой с самого начала.
