Псевдонимы типов (type)

Ключевое слово type позволяет дать короткое имя любому типу — и не только объекту, но и числу, строке, объединению, кортежу или функции. Такое имя называют псевдонимом (по-английски type alias). В этом уроке разберём, как его создавать, зачем это нужно и в чём главная сила type по сравнению с интерфейсом.

Что такое псевдоним типа

Идея простая: слева пишем новое имя, ставим знак =, а справа — тот тип, которому это имя даём. С этого момента имя и исходный тип означают одно и то же — их можно использовать как синонимы.

Аналогия. Псевдоним — как прозвище или сокращение. У человека есть полное имя «Александр», но друзья зовут его «Саша» — это тот же человек, просто короче и привычнее. Точно так же ID — это «прозвище» для string: удобнее читать, а по сути перед нами всё та же строка.

Важно. Псевдоним похож на переменную, но для типа, а не для значения. Переменная const x = 5 даёт имя значению, а type ID = string даёт имя типу. И там, и там есть знак = — это и отличает type от interface, у которого знака = нет.

Знак =: главное отличие от interface

Интерфейс описывает форму объекта без знака =: сразу пишут тело в фигурных скобках. А type — это именно присваивание имени, поэтому знак = обязателен. Сравните:

type для формы объекта

Чаще всего type применяют, чтобы назвать форму объекта — какие поля в нём есть и какого они типа. В этой роли псевдоним работает практически так же, как интерфейс: имя можно ставить в аннотациях переменных, параметров и возвращаемых значений.

Подробнее про синтаксис формы объекта — на странице Типы объектов. Но type умеет и то, чего форма объекта не покрывает совсем.

Сила type: имя для любого типа

Вот где type обгоняет interface. Интерфейс умеет описывать только объекты. А псевдоним можно навесить на что угодно: на примитив, на объединение, на кортеж, на тип функции. Посмотрим на эту универсальность на схеме.

Имя для примитива

Можно дать говорящее имя даже обычному number или string. Код становится понятнее, хотя под капотом остаётся всё тот же примитив.

Имя для объединения — самое полезное

На практике чаще всего псевдоним дают объединению (union) — набору из нескольких допустимых значений через |. Один раз описали список статусов — и переиспользуете имя везде. Интерфейс объединение назвать не может в принципе.

Подробно про объединения и как с ними работать — на странице Объединения (union).

Имя для кортежа

Кортеж — это массив фиксированной длины, где у каждой позиции свой тип. Псевдоним делает такой тип читаемым: Point вместо непонятного [number, number] прямо в аннотации.

Имя для типа функции

Тип функции описывает, что она принимает и что возвращает. Дать ему имя особенно удобно, когда один и тот же вид колбэка встречается в коде много раз.

Композиция: пересечение через &

Псевдонимы можно складывать. Оператор & (пересечение) создаёт тип, у которого есть поля сразу нескольких типов. Читается как «и то, и другое одновременно».

Аналогия. Пересечение & — как «собрать анкету из двух бланков». Был бланк с полями { name, age }; берём второй бланк с полем { role } — и в итоговой анкете нужно заполнить все поля обоих. Пропустишь role — анкета неполная, TypeScript её не примет.

Псевдоним исчезает в рантайме

Важно понимать: type — это исключительно инструмент проверки. Компилятор использует псевдонимы, чтобы найти ошибки, а затем стирает их. В готовом JavaScript-файле никаких type нет — только значения.

Почему так. TypeScript не добавляет проверок типов в готовый JavaScript — иначе программа стала бы медленнее, а браузер вообще не понимает слово type. Поэтому вся типизация — это «леса», которые убирают после стройки: они помогли собрать дом правильно, но в самом доме их уже нет.

Что умеет type, а что interface

Коротко о разнице возможностей. Для формы объекта подходят оба; но как только нужно назвать примитив, объединение, кортеж или функцию — работает только type.

Задача type interface
Форма объекта { name; age } да да
Имя примитиву type Age = number да нет
Объединение 'idle' | 'done' да нет
Кортеж [number, number] да нет
Тип функции (e) => void да нет

Практика

Соберём всё вместе на трёх маленьких, но жизненных примерах: статус запроса, координата с обработчиком и связка «пользователь + админ».

Кратко

  • Псевдоним — короткое имя для типа: type ID = string. У type есть знак =, а у интерфейса — нет.
  • Для формы объекта type работает как тип объекта: type User = { name: string }.
  • Сила type — имя можно дать чему угодно: примитиву (type Age = number), объединению ('idle' | 'done'), кортежу ([number, number]), функции ((e) => void).
  • Композиция через &: type Admin = User & { role: string } — поля обоих типов сразу.
  • Псевдоним стирается в рантайме — это только имя на этапе проверки, в JavaScript его нет.
  • Что выбрать — type или interface — разобрано на странице interface или type.
