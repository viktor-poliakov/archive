TypeScript/Базовые типы/Примитивы

Примитивы

Примитивы — это самые простые «кирпичики» данных: одно число, одна строка, одно «да/нет». В TypeScript их семь, и у каждого есть имя типа, которое можно поставить рядом с переменной. В этом уроке разберём, как ставить такие аннотации типов и какие примитивы бывают.

Что такое аннотация типа

В обычном JavaScript переменная может хранить что угодно и в любой момент менять «сорт» данных. TypeScript позволяет заранее пообещать, что именно будет лежать в переменной. Это обещание и есть аннотация типа — двоеточие и имя типа сразу после имени переменной: : number.

Аналогия. Тип — как ярлык на коробке. Он не влияет на то, что уже лежит внутри, но честно сообщает, что там должно быть. Написали ярлык «числа» — и класть туда носки (строку) больше нельзя: TypeScript сразу заметит несоответствие и подчеркнёт ошибку ещё до запуска программы.

Аннотации у функций

У функции типы ставят в двух местах: у каждого параметра и у возвращаемого значения (после круглых скобок). Так по одной сигнатуре сразу видно, что функция принимает и что отдаёт.

Семь примитивов

Примитив хранит одно простое значение и не имеет «внутренностей», которые можно менять по частям. В TypeScript таких типов семь. Вот их карта:

number — числа

Как и в JavaScript, здесь один числовой тип и для целых, и для дробных — отдельного int нет. Важно: особые значения NaN («не число») и Infinity («бесконечность») тоже входят в number.

bigint — большие целые

Обычный number надёжно хранит целые примерно до 9×10¹⁵. Для больших целых есть bigint: его литералы пишут с суффиксом n. Смешивать bigint и number в одной арифметике нельзя — придётся сначала привести один к другому.

string — текст

Любой текст: одинарные, двойные или шаблонные кавычки `...`. Шаблонные удобны для подстановки значений внутрь строки.

boolean — да/нет

Ровно два значения: true и false. Число 1 «истинно» в условии if, но это не то же самое, что значение true, — присвоить его в boolean нельзя.

symbol — уникальный ключ

Каждый вызов Symbol() даёт новое уникальное значение, даже если описание одинаковое. Символы используют как ключи свойств, которые гарантированно не столкнутся с другими.

null и undefined — пустота

Это два отдельных типа, у каждого ровно одно значение. undefined — «значение ещё не присвоили», null — «мы намеренно поставили пусто».

Строчные имена, а не заглавные

Имена примитивных типов пишутся со строчной буквы: number, string, boolean. Есть похожие имена с заглавной — Number, String, Boolean, — но это другое: типы объектов-обёрток. Как тип переменной их использовать не нужно.

Примитив — пишем так Обёртка — так НЕ пишем
numberNumber
stringString
booleanBoolean

Почему так. Заглавные Number и String — это типы объектов-обёрток, тех самых, в которые движок на миг заворачивает примитив, когда вы зовёте метод вроде .toFixed(). Обёртка «шире» примитива: примитив в неё поместится, а вот обратно объект-обёртку в примитив уже нет. Плюс это сбивает с толку читателя. Правило простое: всегда пишите строчные имена.

null и undefined при строгом режиме

В новых проектах включают флаг strict (а с ним — strictNullChecks). В этом режиме null и undefined — самостоятельные типы, которые не «прячутся» внутри других. Нельзя незаметно положить null в переменную типа string.

Если пустое значение допустимо, разрешите его явно через объединение типов string | null. Тогда компилятор будет напоминать проверять «а не пусто ли тут» перед использованием. Это одна из главных причин, ради которых берут TypeScript: он не даёт забыть про пустые значения.

Без strict проверка отключена, и null с undefined тихо влезают куда угодно — как в обычном JavaScript. Именно поэтому strict рекомендуют держать включённым с самого начала.

Тип часто можно не писать

Аннотация нужна не всегда. Если вы сразу присваиваете значение, TypeScript сам выведет тип — по тому, что вы положили в переменную. Дублировать очевидное вручную не нужно.

Явная аннотация полезна там, где значения при объявлении ещё нет, или когда хочется задать тип шире выведенного (например, string | null). Подробно про то, как именно работает вывод и когда на него полагаться, — на странице «Вывод типов».

Кратко

  • Аннотация типа — : тип после имени: let count: number = 0. У функций аннотируют параметры и возврат: function f(x: string): number.
  • Семь примитивов: number, bigint, string, boolean, symbol, null, undefined.
  • number — и целые, и дробные; NaN и Infinity тоже входят в него.
  • Имена строчные (number), а не заглавные (Number — это объект-обёртка, как тип не используем).
  • При strict null и undefined не прячутся в других типах; чтобы разрешить пусто, пишите string | null.
  • Тип часто можно не писать — TypeScript выведет сам (см. «Вывод типов»).
