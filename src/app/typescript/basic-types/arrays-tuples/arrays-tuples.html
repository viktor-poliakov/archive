Массивы и кортежи

Массивы и кортежи — два способа хранить несколько значений в одной переменной. Разница между ними — в дисциплине: массив держит сколько угодно значений одного типа, а кортеж — строго заданное число значений, у каждого из которых свой тип на своей позиции.

Новичок часто путает их или пишет any[] там, где TypeScript мог бы поймать ошибку. Разберём оба типа подробно: синтаксис, типичные ошибки и реальные сценарии — от пары «ключ-значение» до возврата нескольких значений из функции, как в React-хуке useState.

Главная идея: массив против кортежа

Прежде чем углубляться в синтаксис, поймём разницу на картинке. Массив — это ряд одинаковых ячеек: все они одного типа, а сколько их — не важно, ряд можно удлинять и укорачивать. Кортеж — это несколько разных по назначению ячеек: их ровно столько, сколько объявлено, и каждая ждёт свой тип.

Аналогия. Массив — это железнодорожный состав из одинаковых вагонов. Все вагоны на одно лицо, а сколько их прицепить — дело десятое: можно два, можно двадцать. Кортеж — это конкретные пронумерованные места с назначением: место 1 — водитель, место 2 — штурман, место 3 — механик. Мест ровно столько, сколько предусмотрено, и на каждом сидит «свой» человек — водителя на место штурмана не пересадишь.

Массив: number[] и Array<number>

Тип массива записывается двумя равноправными способами. Первый, короткий и самый частый, — имя типа плюс квадратные скобки: number[] читается как «массив чисел». Второй — через дженерик Array<число>: Array<number>. Это одно и то же, выбирайте по вкусу; в реальных проектах чаще пишут number[].

Ключевое свойство массива: элементов может быть сколько угодно (ноль, один, тысяча), но все они одного типа. Тип как бы говорит: «внутрь можно класть только числа — а сколько именно, решай сам». Попытка положить значение другого типа — ошибка ещё на этапе компиляции, до запуска кода.

Аналогия. number[] — это ящик с наклейкой «только гвозди». Сколько гвоздей насыпать — ваше дело, но шуруп туда класть нельзя: наклейка запрещает. TypeScript — это кладовщик, который проверяет наклейку при каждой попытке что-то положить.

Массив из нескольких типов: (string | number)[]

А если в списке должны лежать значения разных типов вперемешку? Тогда в качестве «типа элемента» берут объединение (union) — несколько типов через |. Запись (string | number)[] означает «массив, каждый элемент которого — либо строка, либо число».

Внимание — самая частая ошибка. Скобки здесь несут смысл. (string | number)[] — это массив из строк и чисел. А string | number[] без скобок — это «либо одна строка, либо массив чисел». TypeScript читает [] как относящееся только к ближайшему number. Если сомневаетесь — ставьте скобки.

  • (string | number)[] — массив, где вперемешку строки и числа.
  • string | number[] — либо строка, либо массив чисел (это не массив строк!).
  • string[] | number[] — либо массив только строк, либо массив только чисел.

readonly-массивы: только для чтения

Обычный массив можно менять: добавлять, удалять, перезаписывать элементы. Иногда это опасно — например, вы передали массив в функцию и не хотите, чтобы она его испортила. Тогда объявляют массив только для чтения: readonly number[] или, что то же самое, ReadonlyArray<number>.

У такого типа исчезают все мутирующие методы (push, pop, splice, sort) и запрещена запись по индексу. Читать — arr[0], arr.length, перебор в цикле — можно свободно. Это отличный способ задокументировать намерение «я эти данные только читаю» и заставить компилятор следить за этим за вас.

Аналогия. readonly number[] — это музейная витрина. Экспонаты видно прекрасно, пересчитать можно, а вот взять руками или переставить — стекло не даст. Немутирующие методы (map, filter, toSorted) при этом работают: они не трогают витрину, а делают новую копию снаружи.

Кортеж: [string, number]

Кортеж (tuple) — это массив с заранее известной длиной, где на каждой позиции свой тип. Пишется в квадратных скобках через запятую: [string, number] — «ровно два элемента: сначала строка, потом число». Порядок фиксирован: строка обязана быть первой, число — вторым.

TypeScript помнит тип каждой позиции. Обращение user[0] даёт string, а user[1] — number: приводить их к общему типу не нужно, компилятор всё знает точно. Зато он же строго следит за длиной и порядком:

Именованные элементы кортежа

Голый [string, number] не подсказывает, что за строка и что за число. Чтобы код читался яснее, позициям дают имена: [name: string, age: number]. На поведение кода это никак не влияет — имена нужны исключительно для читаемости и подсказок редактора.

Имена особенно ценны, когда кортеж описывает параметры функции (через ...rest): в подсказках IDE аргументы будут подписаны осмысленно, а не безликими arg_0, arg_1. Это дешёвая документация прямо в типе.

Опциональные ? и rest-элементы

Длина кортежа не обязана быть жёстко зафиксированной до последнего элемента. Есть два инструмента гибкости.

Опциональные элементы: ?

Знак ? после типа делает хвостовой элемент необязательным. [number, number, number?] — это «два числа обязательно, третье по желанию». Тип необязательного элемента автоматически включает undefined, ведь его может и не быть.

rest-элементы: ...

Оператор ... позволяет закрепить первые позиции, а дальше принять сколько угодно значений одного типа. [string, ...number[]] — «сначала одна строка-метка, затем ноль или больше чисел». Это соединяет строгость кортежа (первая позиция точно строка) с растяжимостью массива (чисел может быть любое количество).

Правило хвоста. И ?, и ... относятся к концу кортежа: обязательный элемент нельзя поставить после необязательного, а после rest-элемента обычно уже ничего не идёт. Логика простая — TypeScript должен однозначно понимать, какая позиция какому типу соответствует.

Практика: где это встречается каждый день

Пара «ключ-значение»

Кортеж из двух элементов — естественная форма для пары «ключ → значение». На таких парах построены Object.entries (он возвращает массив кортежей [string, значение]) и конструктор Map.

Возврат нескольких значений из функции (как useState)

Функция умеет вернуть только одно значение — но если это кортеж, внутри него можно «упаковать» несколько разнотипных результатов. Ровно так работает хук useState в React: он возвращает [значение, функцию-сеттер], а вы деструктурируете их по позициям и даёте свои имена.

Почему кортеж, а не объект. Кортеж хорош тем, что имена вы придумываете на месте: const [count, setCount] и const [name, setName] из одной и той же функции. С объектом пришлось бы каждый раз переименовывать поля. Поэтому «пара с деструктуризацией по позициям» — идиома для функций вроде useState.

Кратко

  • Массив — любое число элементов одного типа: number[] или эквивалент Array<number>.
  • Несколько типов в массиве — через объединение в скобках: (string | number)[]. Скобки обязательны: string | number[] — совсем другой тип.
  • readonly-массив (readonly number[] / ReadonlyArray<number>) запрещает мутации: ни push, ни запись по индексу.
  • Кортеж [string, number] — фиксированные длина и типы по позициям; порядок важен.
  • Имена элементов ([name: string, age: number]) улучшают читаемость, не меняя поведения.
  • ? делает хвостовые элементы необязательными, ... добавляет растяжимый хвост: [string, ...number[]].
  • Кортежи — это пары «ключ-значение» и возврат нескольких значений из функции (как useState).

Дальше — any и unknown, а если хочется вернуться к одиночным значениям — загляните в Примитивы. Как TypeScript сам угадывает типы массивов и кортежей, разбирается в разделе Вывод типов.
