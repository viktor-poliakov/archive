Приведение типов

JavaScript — язык со слабой типизацией: он охотно сам приводит один тип к другому, когда операция этого требует. Приведение бывает явным (мы вызываем Number(), String(), Boolean()) и неявным (происходит автоматически внутри операторов). Всего три направления: к boolean, к number и к string.

Явное преобразование

Самый понятный случай — мы сами просим преобразовать значение, вызвав функцию. Детали работы Number() и parseInt разобраны в разделе Числа: методы; здесь важна общая картина всех трёх направлений.

К boolean

Приведение к boolean случается в условиях (if, while), в операторе ! и в логических операторах. Запомнить нужно ровно 8 falsy-значений — всё остальное истинно (полный список — в разделе Типы). Главные сюрпризы: строка '0', пустой массив и пустой объект — все truthy.

К number

К числу значение приводится в арифметике (кроме особого +), в сравнениях и при унарном +. Правила по большей части интуитивны, но массивы и объекты дают неожиданные результаты: пустой массив это 0, массив из одного числа — само число, а из нескольких — уже NaN.

К string

К строке значение приводится оператором + рядом со строкой, функцией String() и внутри шаблонных строк. Массив склеивается через запятую, а обычный объект превращается в малополезную строку '[object Object]'.

Объекты → примитивы

Когда объект попадает в операцию, ожидающую примитив, он сначала преобразуется — как правило, через toString(). Отсюда знаменитые «странности» вроде [] + {}. Управлять этим можно методами valueOf, toString и символом Symbol.toPrimitive.

Сводная таблица

Как популярные значения приводятся к каждому из трёх типов:

Значение Number() String() Boolean()
'' 0 '' false
'0' 0 '0' true
0 0 '0' false
null 0 'null' false
undefined NaN 'undefined' false
[] 0 '' true
[5] 5 '5' true
{} NaN '[object Object]' true

Осторожно: пустой массив [] — это 0 как число, но true как boolean. Такие расхождения — частый источник багов; лучше приводить типы явно (см. Нюансы).

Кратко

  • Три направления: к boolean, number, string; приведение бывает явным и неявным.
  • К boolean: 8 falsy-значений, всё прочее truthy (в т. ч. '0', [], {}).
  • К number: ''→0, '12px'→NaN, []→0, [5]→5, объект→NaN.
  • К string: массив через запятую, объект → '[object Object]'.
  • Не полагайтесь на неявное — приводите типы явно, это надёжнее и понятнее.
