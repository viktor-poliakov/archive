Операторы сравнения

Операторы сравнения возвращают boolean — true или false. Главное, что нужно усвоить: разница между строгим (===) и нестрогим (==) равенством. От неё зависит, приводятся ли типы перед сравнением.

Строгое равенство ===

=== (и обратный ему !==) сравнивает без приведения типов: значения считаются равными, только если у них совпадают и тип, и значение. Это предсказуемо, поэтому === берут по умолчанию.

Нестрогое равенство ==

== (и !=) перед сравнением приводит типы по хитрым правилам, из-за чего часто даёт неочевидные результаты. Общая рекомендация — избегать ==. Единственное разумное применение — проверка x == null, которая разом ловит и null, и undefined.

Сравнение == (приводит типы) === (без приведения)
5 и '5' true false
0 и '' true false
null и undefined true false

Обзор == vs === есть и в разделе Типы. Здесь мы смотрим на них как на часть системы операторов.

Реляционные < > <= >=

Операторы «больше/меньше» сравнивают числа по величине. Но со строками работают лексикографически — посимвольно по кодам символов, а не по длине. Отсюда знаменитое '2' > '10' — это true. Если операнды разного типа, они приводятся к числу.

Подробно про сравнение строк и порядок символов (Unicode, localeCompare) — в разделе Строки: Юникод.

Особые значения: null, undefined, NaN

Три значения ведут себя в сравнениях по-своему. null и undefined нестрого равны друг другу, но строго — нет. А NaN не равен ничему, включая самого себя, — поэтому проверять его сравнением бесполезно (только Number.isNaN, см. Числа: нюансы).

Кратко

  • === по умолчанию — сравнивает тип и значение без приведения.
  • == приводит типы и даёт сюрпризы; оправдан лишь как x == null.
  • Реляционные сравнивают числа по величине, строки — лексикографически ('2' > '10').
  • Особые: null == undefined (true), а NaN не равен ничему — берите Number.isNaN.
