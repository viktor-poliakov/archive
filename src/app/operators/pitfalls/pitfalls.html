Операторы: нюансы и подводные камни

Собрали самые коварные грабли операторов. Почти все они — следствие неявного приведения типов и приоритета операторов. Общий рецепт против большинства из них: сравнивать через === и приводить типы явно.

Ловушки ==

Нестрогое равенство полно исключений, которые невозможно запомнить логикой. Самая известная пара: null == 0 — это false, но null >= 0 — уже true (потому что >= приводит null к числу, а == — нет). Именно поэтому == избегают.

+ зависит от порядка

Поскольку + и складывает, и склеивает, результат зависит от того, где в цепочке встретилась первая строка. Вычисление идёт слева направо, и как только появляется строка — весь остаток превращается в конкатенацию.

Приоритет операторов

Как и в математике, * и / выполняются раньше + и -; && связывает сильнее, чем ||. Когда сомневаетесь — ставьте скобки: они и меняют порядок, и делают код понятнее. Особый случай: ?? нельзя смешивать с ||/&& без скобок — это синтаксическая ошибка.

Числовые ловушки

Часть граблей приходит из чисел: NaN не равен самому себе, а 0.1 + 0.2 не равно 0.3. Это разобрано в разделах Числа: нюансы и Точность и большие числа.

Кратко

  • Не используйте ==: null >= 0 это true, а null == 0 — false. Берите ===.
  • + зависит от порядка: первая строка превращает остаток в конкатенацию (1 + 2 + '3' → '33').
  • Скобки решают: *// раньше +/-; ?? нельзя мешать с ||/ && без скобок.
  • Числа: NaN !== NaN, 0.1 + 0.2 !== 0.3 — см. раздел «Числа».
