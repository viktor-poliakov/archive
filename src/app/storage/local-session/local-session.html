localStorage и sessionStorage

Это Web Storage — самое простое хранилище: пары «ключ-значение», обе части — строки. Есть два близнеца с одинаковым API: localStorage хранит данные без срока, а sessionStorage — только пока открыта вкладка.

Только строки — объекты через JSON

Главное ограничение: Web Storage хранит только строки. Число молча превратится в строку (42 → '42'), а объект без подготовки — в бесполезное '[object Object]'. Поэтому объекты сохраняют через JSON:

При разборе оборачивайте JSON.parse в try / catch: в хранилище может лежать испорченная строка, и разбор бросит ошибку.
