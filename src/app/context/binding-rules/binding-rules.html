Правила и приоритет

«Объект перед точкой» — самый частый случай, но всего функцию можно вызвать четырьмя способами, и у каждого своё правило для this. Зная все четыре и их порядок, можно определить this в любой ситуации.

Четыре правила привязки

  • По умолчанию — fn(), вызов без точки. this равен undefined (в строгом режиме) или глобальному объекту (см. «Привязка по умолчанию»);
  • Неявная — obj.method(), есть объект перед точкой. this равен этому объекту;
  • Явная — fn.call(obj) / fn.apply(obj) / fn.bind(obj). Мы сами назначаем this (см. «call, apply, bind»);
  • Через new — new Fn(). Создаётся новый объект, и this указывает на него (см. «this в классах и new»).

Приоритет правил

Если к одному вызову подходит несколько правил, побеждает более «сильное». Порядок от сильного к слабому:

new > явная (bind/call/apply) > неявная (obj.method()) > по умолчанию (fn()).

Краевой случай: bind + new

Когда привязанную через bind функцию вызывают с new, new побеждает: this станет новым объектом, а не тем, что был привязан. При этом предзаполненные аргументы из bind всё равно применяются:

Как быстро определить this

Идите сверху вниз — первое подходящее правило и есть ответ:

  • функция стрелочная? → this из внешнего окружения (своего нет, см. «Стрелочные функции»);
  • вызвана с new? → новый объект;
  • есть call/apply/bind? → указанный там объект;
  • есть объект перед точкой? → этот объект;
  • ничего из этого? → undefined (strict) или globalThis.
