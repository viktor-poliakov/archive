Стрелочные функции и this

Стрелочные функции ведут себя с this принципиально иначе, чем обычные. Понимание этого отличия снимает половину вопросов про this.

У стрелки нет своего this

Обычная функция при вызове получает свой this по правилам привязки. У стрелочной функции своего this нет вообще — она берёт его из окружающего кода, там где была написана (это называют «лексическим» this). Поэтому стрелки идеальны для колбэков внутри методов:

Ловушка: стрелка как метод объекта

Из той же причины вытекает классическая ошибка. Если сделать метод объекта стрелочным, this будет указывать не на объект, а на внешнюю область (в модуле — undefined, в обычном скрипте — window):

Правило: методы объекта пишите обычными (сокращёнными) функциями, а стрелки используйте для колбэков внутри этих методов.

Обычная против стрелочной

Свойство Обычная функция Стрелочная
Свой this есть (по правилам вызова) нет (берёт из окружения)
Можно сменить через bind/call да нет
Годится как метод объекта да нет (ловушка)
Годится как колбэк внутри метода теряет this да
Можно вызвать с new да нет
Свой arguments есть нет

Кроме this, у стрелок нет своего arguments, нет prototype и их нельзя вызвать через new — поэтому как методы класса или конструкторы они не подходят (см. «this в классах и new»).
