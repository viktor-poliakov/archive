Цепочка и практика

Из отдельных ссылок __proto__ складывается цепочка прототипов, по которой движок ищет свойства. Разберём полную цепочку, как на ней устроены встроенные объекты, что делает instanceof и какими методами со всем этим работать.

Прототипы — не экзотика, на них держится весь язык. Методы массива лежат не в каждом массиве, а один раз в Array.prototype, общем для всех:

Обратите внимание на две последние строки: сам Array.prototype — тоже объект, поэтому его прототип — Object.prototype. А у Object.prototype прототипа уже нет: его __proto__ равен null — это и есть вершина любой цепочки.

Методы для работы с прототипами

Современный набор инструментов вместо устаревшего __proto__:

  • Object.create(proto) — создать объект с заданным прототипом.
  • Object.getPrototypeOf / Object.setPrototypeOf — прочитать / поменять прототип.
  • Object.hasOwn(obj, key) — свойство собственное, а не унаследованное.
  • proto.isPrototypeOf(obj) — есть ли proto в цепочке объекта.

Подводные камни

  • Не меняйте нативные прототипы (Object.prototype, Array.prototype): свойство появится у всех объектов и сломает перебор и чужой код.
  • __proto__ устарел — используйте Object.getPrototypeOf/setPrototypeOf.
  • Object.setPrototypeOf — медленный; задавайте прототип при создании (Object.create, class), а не меняйте потом.
  • При переборе чужих объектов проверяйте Object.hasOwn, чтобы не задеть унаследованное.

Кратко

  • __proto__ / [[Prototype]] — ссылка объекта вверх по цепочке; есть у любого объекта.
  • prototype — свойство функции/класса, «справочник» для будущих экземпляров.
  • Связь: new F() ⇒ obj.__proto__ === F.prototype.
  • Цепочка заканчивается на Object.prototype, затем null; методы ищутся снизу вверх.
  • Работать через Object.create/getPrototypeOf; нативные прототипы не трогать.
