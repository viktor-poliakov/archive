Оператор new и конструкторы

Мы уже знаем главное равенство темы: new F() ⇒ экземпляр.__proto__ === F.prototype. Теперь разберём полную картину: что new делает по шагам и почему конструктор может вернуть совсем другой объект.

Функция-конструктор

Конструктор — это обычная функция, которую вызвали через new. По соглашению её имя пишут с большой буквы. Внутри доступен this — новый объект, который создал new; его нужно лишь дозаполнить свойствами.

Что делает new по шагам

Оператор new делает больше, чем «просто вызывает функцию». Сначала он сам готовит объект, и только потом отдаёт его конструктору на дозаполнение. Всю его работу можно записать псевдокодом:

  • 1. создаётся новый пустой объект, и его __proto__ сразу ставится на Constructor.prototype;
  • 2. вызывается конструктор с this, равным этому объекту;
  • 3. если конструктор вернул объект — результатом будет он, иначе — тот объект из шага 1.

Ключевой вывод: объект создаётся до тела конструктора, а не конструктором. Конструктор получает уже готовый (но пустой) объект как this и просто дозаполняет его — в 99% случаев он ничего не создаёт с нуля.

Возврат из конструктора

Тот самый шаг 3 — самый неожиданный. Если внутри конструктора есть return, правило такое:

  • Вернули примитив (число, строку, boolean) или ничего — return игнорируется, вы получаете обычный this.
  • Вернули объект — именно он станет результатом new F(), а this отбрасывается.

Правило про __proto__ не нарушается. Просто в этом случае __proto__ указывает на прототип того объекта, который реально вернули (у литерала {} это Object.prototype), а не на Animal.prototype. То есть new гарантирует связку __proto__ = F.prototype только когда конструктор не подменил результат явным return объекта.

Условный и явный return

Особой синтаксической конструкции для «условного возврата» не существует — это обычный if в теле функции. JavaScript в конце просто смотрит, что вернул конструктор: объект или нет.

А если вызвать без new?

Тогда this — уже не новый объект. В строгом режиме (а в модулях и классах он включён всегда, см. Контекст) this равен undefined, и первое же this.x = … бросит TypeError — это спасает от частой ошибки новичков. В нестрогом режиме было бы хуже: свойства «утекли» бы в глобальный объект.

С классами проще: вызов класса без new сразу бросает TypeError — язык не даёт случайно потерять this. Подробнее — в разделе this в классах и new.

Кратко

  • Конструктор — обычная функция, вызванная через new; this — готовый объект для дозаполнения.
  • new по шагам: создать объект и связать его __proto__ с F.prototype → вызвать конструктор с этим this → вернуть его (или объект из return).
  • Объект создаётся до тела конструктора — тот его лишь дозаполняет.
  • Возврат: примитив игнорируется; объект подменяет this (и __proto__ будет от него).
  • Паттерны: синглтон и кэш инстансов — через return готового объекта.
  • Без new: у функции — TypeError в strict; у класса — TypeError всегда.
