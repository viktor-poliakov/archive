Свой итератор

Мы видели, что обычный объект не итерируем. Исправим это: научим свой объект работать с for...of и spread. Для этого достаточно добавить ему метод [Symbol.iterator], который возвращает итератор с next().

Объект-диапазон

Сделаем объект range, который перебирает числа от from до to. Метод [Symbol.iterator]() заводит счётчик и возвращает итератор; каждый next() выдаёт следующее число, а когда числа кончаются — сообщает done: true.

Тонкость: счётчик current живёт внутри [Symbol.iterator](), а не на самом объекте. Поэтому каждый новый перебор начинается с начала и можно спокойно запускать несколько for...of одновременно — у каждого свой независимый итератор. (Это тот же приём приватного состояния через замыкание.)

Теперь работает всё

Как только у объекта появился [Symbol.iterator], он стал полноценным iterable — и с ним заработали все потребители протокола: цикл, spread, Array.from, передача через ....

Кратко

  • Чтобы объект стал итерируемым, добавьте метод [Symbol.iterator]().
  • Он возвращает итератор — объект с next(), который отдаёт { value, done }.
  • Состояние перебора держите внутри (в замыкании), чтобы каждый перебор был независимым.
  • Писать это вручную многословно — генераторы делают то же самое в разы короче (следующий раздел).
