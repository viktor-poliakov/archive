for...of и потребители протокола

Хорошая новость: раз есть единый протокол итерации, вам почти никогда не придётся звать next() руками. Множество конструкций языка умеют «потреблять» любой iterable — и главная из них — for...of.

Цикл for...of

for...of перебирает значения итерируемого — без возни с индексами. Работают привычные break и continue. Если нужен ещё и индекс, у массива есть entries().

Не путайте с for...in: for...in перебирает ключи объекта (и не гарантирует порядок), а for...of — значения итерируемого. Подробнее про перебор массивов — в разделе Массивы: перебор.

Потребители протокола

for...of — не единственный. Один и тот же протокол используют spread ..., деструктуризация, Array.from, конструкторы new Set(...)/new Map(...), передача аргументов через f(...it) и даже Promise.all(iterable). Научились делать что-то итерируемым — и всё это заработает с ним автоматически.

Кто итерируем из коробки

Готовые iterable в языке: Array, String, Map, Set, а также arguments и коллекции DOM (NodeList). А вот обычный объект { a: 1 } — не итерируем: у него нет [Symbol.iterator]. Для него перебирают Object.keys/values/entries, либо делают объект итерируемым самому — об этом следующий раздел.

Кратко

  • for...of перебирает значения любого iterable; break/continue работают.
  • Тот же протокол используют spread, деструктуризация, Array.from, new Set/Map, Promise.all.
  • Итерируемы: Array, String, Map, Set, arguments, NodeList.
  • Обычный объект — нет; для него — Object.entries или свой [Symbol.iterator].
