JavaScript / Типы

Типы данных

У каждого значения в JavaScript есть тип — «сорт» данных: число, строка, «да/нет» и так далее. Тип подсказывает, что со значением можно делать: числа складывать, строки склеивать. Разберём, какие типы бывают и почему про JavaScript говорят, что у него динамическая типизация.

Динамическая типизация

Главная особенность: в JavaScript тип привязан к значению, а не к переменной. Одна и та же переменная может сегодня хранить строку, а через строчку — число, и это не ошибка:

Это и есть динамическая типизация: тип определяется прямо во время выполнения, по текущему значению. Для сравнения, в языках со статической типизацией (TypeScript, Java) тип переменной фиксируется заранее, и положить в неё значение другого типа нельзя.

Какие типы бывают

Всего типов восемь: семь примитивных и один сложный — объект. Примитив хранит одно простое значение, объект — целую коллекцию данных.

Про null и undefined: undefined означает «значение ещё не присвоено» (так выглядит объявленная, но не заполненная переменная), а null — «намеренно пусто», когда мы сами хотим сказать «здесь ничего нет».

Обёртки примитивов

Возникает вопрос: если 'hello' — примитив, откуда у него .length и методы вроде .toUpperCase()?

Секрет в обёртках: при обращении через точку движок на миг оборачивает примитив в специальный объект (String, Number, Boolean), берёт нужный метод и тут же выбрасывает обёртку. Поэтому методы «есть», а лишних объектов не остаётся. Создавать обёртки вручную через new String(...) не нужно.

Числа: NaN, Infinity, точность

У числового типа есть несколько особых значений и подводных камней:

  • NaN («не число») — результат некорректной операции; он не равен даже сам себе, проверяют через Number.isNaN().
  • Infinity — «бесконечность», например при делении на ноль.
  • Дробные числа хранятся приближённо, поэтому 0.1 + 0.2 не равно ровно 0.3.
  • bigint нужен для целых чисел, которые больше безопасного предела обычного number.

Это лишь вершина айсберга. Подробно про системы счисления, Math, преобразование, точность и BigInt — в разделе Числа и Math.

Преобразование типов

Раз тип определяется значением, JavaScript часто сам приводит один тип к другому. Приведение бывает явным (мы просим) и неявным (происходит автоматически).

truthy и falsy: в условиях (if) любое значение приводится к boolean. «Ложных» значений всего восемь: false, 0, -0, 0n, '' (пустая строка), null, undefined, NaN. Всё остальное — «истинно» (в том числе строка '0' и пустой объект).

Кратко

  • Восемь типов: 7 примитивов (number, bigint, string, boolean, null, undefined, symbol) + object.
  • Динамическая типизация: тип принадлежит значению, переменная может хранить любой тип.
  • typeof узнаёт тип; помните про typeof null === 'object'.
  • Примитивы — по значению, объекты — по ссылке.
  • Сравнивайте через ===, а не ==, чтобы избежать неявных приведений.
