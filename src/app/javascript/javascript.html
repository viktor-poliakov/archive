JavaScript

JavaScript — самый распространённый язык программирования в вебе. Изначально он был придуман, чтобы «оживлять» страницы в браузере, но сегодня на нём пишут серверы, мобильные и десктопные приложения, и даже управляют устройствами.

Немного истории

Язык создал Брендан Айк (Brendan Eich) в компании Netscape в 1995 году — по легенде, первый прототип он собрал примерно за 10 дней. Сначала язык назывался Mocha, затем LiveScript, а финальное имя JavaScript выбрали из маркетинговых соображений — на волне популярности языка Java. Несмотря на схожие названия, Java и JavaScript — это совершенно разные языки.

Чтобы язык работал одинаково в разных браузерах, его стандартизировали. Стандарт называется ECMAScript (спецификация ECMA-262), а «JavaScript» — это его самая известная реализация. Поэтому версии языка обозначают как ES: например, ES5, ES6.

Ключевые вехи развития:

  • 1997 — первый стандарт ECMAScript;
  • 2009 — ES5: язык стал зрелее (строгий режим 'use strict', методы массивов). В том же году появился Node.js — и JavaScript вышел за пределы браузера, на сервер;
  • 2015 — ES6 (ES2015): огромное обновление, во многом современный вид языка — let/const, стрелочные функции, классы, модули, промисы;
  • с 2015 — ежегодные релизы (ES2016, ES2017 с async/await и так далее) — язык развивается небольшими порциями каждый год.

Рядом часто упоминают TypeScript — это надстройка над JavaScript со статической типизацией. В браузере он не выполняется напрямую: его сначала превращают в обычный JavaScript.

Для чего он нужен

Главная сила JavaScript — он работает почти везде. Один язык — множество сред:

  • Фронтенд — единственный язык, который понимают браузеры «из коробки»: реакция на действия пользователя, изменение страницы (DOM), запросы к серверу. На нём построены React, Angular, Vue;
  • Бэкенд — серверы на Node.js, Deno, Bun;
  • Мобильные приложения — React Native;
  • Десктоп — Electron (на нём сделаны VS Code, Slack);
  • Прочее — скрипты, автоматизация, даже микроконтроллеры.

Как движок обрабатывает код

JavaScript не компилируется заранее в отдельный файл, как C++ — код выполняет движок, встроенный в браузер или в Node.js. Современные движки не «тупо интерпретируют» код построчно, а используют JIT-компиляцию (Just-In-Time — «точно в срок»). Упрощённо путь кода такой:

  • Парсинг. Текст программы разбирается на токены, из которых строится дерево — AST (абстрактное синтаксическое дерево);
  • Байт-код. По дереву генерируется промежуточный байт-код, который сразу же выполняет интерпретатор;
  • Оптимизация. Движок следит, какие участки кода выполняются часто («горячий» код), и компилирует их в быстрый машинный код. Если предположения нарушились (например, в функцию вдруг передали не число, а строку) — оптимизация откатывается.

Аналогия. Интерпретатор — как синхронный переводчик, переводящий фразу за фразой на ходу. JIT — переводчик, который заметил, что вы повторяете один и тот же абзац сотни раз, и один раз сделал его качественный письменный перевод, чтобы дальше выдавать мгновенно.

Движки и компиляторы

Движки — это и есть программы, которые выполняют JavaScript. У каждого крупного браузера свой:

Движок Где используется
V8 Chrome, новый Edge, Node.js, Deno
SpiderMonkey Firefox (первый движок JS, тоже от Брендана Айка)
JavaScriptCore Safari, Bun

Отдельно стоят компиляторы и транспиляторы — инструменты, которые работают до запуска и превращают один исходный код в другой (это называют source-to-source компиляцией). Их не стоит путать с движком:

  • Babel — превращает современный JavaScript в более старый, чтобы код работал в устаревших браузерах;
  • TypeScript (tsc) — компилирует TypeScript в обычный JavaScript;
  • esbuild, SWC — очень быстрые сборщики/компиляторы (написаны на Go и Rust), делают то же самое в разы быстрее.

Коротко о разнице: транспилятор (Babel, tsc) переводит код из одного вида в другой заранее; движок (V8) уже во время выполнения интерпретирует и JIT-компилирует то, что получилось, в машинный код.
