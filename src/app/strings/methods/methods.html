Методы работы со строками

У строк есть большой набор встроенных методов: искать, вырезать куски, менять регистр, заменять и разбивать. Главное помнить из основ: строки неизменяемы, поэтому ни один метод не меняет исходную строку — все возвращают новую. Разберём методы по задачам.

Поиск и проверка

Чтобы узнать, есть ли подстрока, читаемее всего includes, startsWith и endsWith — они возвращают boolean. Если нужна позиция, есть indexOf и lastIndexOf; важно помнить, что при отсутствии совпадения они возвращают -1, а не 0.

Извлечение подстроки

Основной инструмент — slice(start, end): возвращает кусок от start до end (не включая его) и понимает отрицательные индексы, отсчитывая с конца. Есть похожий substring, но он отрицательные индексы превращает в 0 и молча меняет аргументы местами, если start > end — поведение неочевидное. Старый substr считается устаревшим.

Вызов slice substring
Отрицательный индекс отсчёт с конца превращается в 0
start > end пустая строка меняет аргументы местами

Совет: в новом коде почти всегда берите slice — он предсказуемее. К substring прибегают редко.

Регистр и обрезка пробелов

toUpperCase и toLowerCase меняют регистр (частый приём перед сравнением без учёта регистра). Семейство trim убирает пробелы по краям — например, из того, что ввёл пользователь.

Замена

Здесь популярная ловушка: replace со строкой-образцом заменяет только первое совпадение. Чтобы заменить все, есть replaceAll или регулярное выражение с флагом g. Вторым аргументом можно передать функцию — она получит совпадение и вернёт замену.

Разбиение и сборка

split(разделитель) превращает строку в массив, а обратная операция — join (уже метод массива). Ещё пара полезных: repeat повторяет строку, а padStart/padEnd дополняют её до нужной длины (удобно для выравнивания и форматирования чисел).

Кратко

  • Методы не меняют строку — всегда возвращают новую.
  • Поиск: includes/startsWith/ endsWith → boolean; indexOf → позиция или -1.
  • Извлечение: берите slice (умеет отрицательные индексы); substring ведёт себя неочевидно.
  • Замена: replace — первое совпадение; replaceAll или /g — все.
  • Разбиение: split ↔ join; плюс repeat, padStart/padEnd.

Регулярные выражения открывают куда более мощный поиск и замену по шаблону — это отдельная большая тема, которая заслуживает своего раздела.
