Всплытие ошибок по стеку

Ошибку не обязательно ловить там, где она возникла. Если рядом нет catch, она всплывает вверх по цепочке вызовов — от функции к той, что её вызвала, — пока не встретит ближайший try/catch.

Аналогия — эскалация вверх по начальству. Рядовой сотрудник не смог решить вопрос — передал руководителю; тот не смог — передал выше. Так до тех пор, пока кто-то не разберётся. А если не разобрался никто на всех уровнях — это уже «авария» для всей компании (необработанная ошибка).

Ошибка всплывает до ближайшего catch

Функция, которая не поймала ошибку, сама прерывается и «отдаёт» ошибку вызвавшей. В примере c() бросает, b() её не ловит (и потому тоже прерывается), а a() — ловит. Обратите внимание: catch в a() перехватывает ошибку, рождённую двумя уровнями глубже.

Необработанная ошибка

Если catch не встретился нигде по всей цепочке, ошибка «выпадает» наружу как необработанная. Это не «тихо потерялось» — среда об этом громко сообщит.

Где ловить: не везде

Раз ошибки всплывают, не нужно оборачивать в try/catch каждую функцию. Ловите там, где можете осмысленно обработать ошибку: показать сообщение пользователю, повторить запрос, подставить значение по умолчанию. Если обработать нечем — пусть всплывает дальше. А если нужно лишь добавить контекст и передать ошибку выше — её пробрасывают повторно через throw.

Кратко

  • Не пойманная ошибка всплывает вверх по стеку вызовов до ближайшего catch.
  • Промежуточные функции прерываются — их код после вызова не выполнится.
  • Никто не поймал — ошибка становится необработанной (консоль / window.onerror / падение процесса).
  • Ловите осмысленно: там, где можете обработать; иначе пусть всплывает или пробрасывайте через throw.
