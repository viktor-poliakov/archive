Объект Error и типы ошибок

Всё, что бросают через throw, по-хорошему — объект Error. Он несёт три вещи: имя типа, сообщение и стек (где произошло). А встроенные подтипы Error подсказывают, что именно пошло не так.

Свойства Error

У любого объекта ошибки есть три ключевых свойства: name — тип, message — человекочитаемое описание (его передают в конструктор), и stack — строка со стеком вызовов на момент ошибки (бесценна при отладке).

Встроенные типы ошибок

JavaScript сам бросает не безликий Error, а конкретный подтип — по нему сразу видно природу проблемы. Знать основные полезно, чтобы понимать сообщения в консоли и осмысленно их обрабатывать.

Тип Когда возникает Пример
TypeError значение не того типа / операция недопустима null.foo
ReferenceError обращение к несуществующей переменной notDefined
RangeError значение вне допустимого диапазона (5).toFixed(200)
SyntaxError невалидный код или данные JSON.parse('{ x }')
URIError неверный вызов функций URI decodeURIComponent('%')

Некоторые из этих типов подробно разобраны в контексте своих тем: ReferenceError — в разделах Hoisting и Лексическое окружение, TypeError при вызове конструктора без new — в Оператор new и конструкторы.

Разбор через instanceof

Раз типы — это классы-потомки Error, их различают оператором instanceof и реагируют по-разному. Проверять важно от частного к общему: instanceof Error истинно для любой ошибки, поэтому эту проверку ставят последней.

Цепочка причин: cause

Часто низкоуровневую ошибку удобно «завернуть» в понятную высокоуровневую — но при этом не потерять исходную причину. Для этого у Error есть второй аргумент { cause }: он сохраняет оригинальную ошибку в свойстве cause.

Живой пример такого приёма — обёртка сетевых ошибок в разделе REST API: обработка ошибок.

Кратко

  • Error несёт name, message и stack.
  • Встроенные типы: TypeError, ReferenceError, RangeError, SyntaxError, URIError — все потомки Error.
  • Разбор — через instanceof, от частного к общему.
  • { cause } сохраняет первопричину при оборачивании ошибок.
