Обработка ошибок: нюансы и подводные камни

Собрали типичные ошибки в работе с ошибками — от «проглатывания» до самой частой путаницы: почему try/catch не ловит асинхронный код.

Не «проглатывайте» ошибки

Пустой catch (или catch без реакции) — худшее, что можно сделать: программа продолжает работать так, будто всё хорошо, а на самом деле что-то сломалось, и вы об этом не узнаете. Как минимум логируйте ошибку; а лучше — обработайте её или пробросьте дальше.

Исключения — не для потока управления

Исключения нужны для внештатных ситуаций, а не для обычной логики. Использовать throw/catch вместо if/ return для ожидаемых веток — плохая идея: код труднее читать, а сами исключения работают медленнее обычных проверок.

finally перекрывает return

Коварная деталь: return (или throw) внутри finally затирает то, что вернул или бросил try/catch. Поэтому finally держат только для «уборки», без return/throw.

Ловите узко, лишнее — пробрасывайте

Не превращайте catch в «ловушку для всего». Обрабатывайте только те ошибки, с которыми знаете что делать (например, по instanceof), а остальные пробрасывайте через throw. Иначе за общим catch легко спрятать настоящий баг.

try/catch НЕ ловит асинхронное

Это главная путаница темы. try/catch ловит только синхронные ошибки, случившиеся во время выполнения try. А колбэк setTimeout, обработчик события или «плавающий» промис выполнятся позже — когда try уже завершился, а стек вызовов стал другим. Ошибка из них мимо этого catch пролетит.

Почему так: отложенный колбэк запускается из событийного цикла с «чистого» стека — см. Event Loop. Для асинхронных ошибок есть свои средства: .catch у промисов и try/catch вокруг await — подробно в разделах Обработка ошибок в промисах и async/await.

Кратко

  • Не глотайте ошибки пустым catch — логируйте, обрабатывайте или пробрасывайте.
  • Не используйте исключения для обычного потока управления.
  • return/throw в finally затирают результат из try/catch — не делайте так.
  • Ловите узко, чужое пробрасывайте дальше.
  • Асинхронные ошибки (setTimeout, промисы) обычный try/catch не ловит — нужны .catch / await.
