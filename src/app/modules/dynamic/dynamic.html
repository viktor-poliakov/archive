Динамический импорт

Обычный import — статический: он пишется в самом верху файла и выполняется всегда, ещё до запуска кода. Но иногда модуль нужен не сразу, а по условию или по действию пользователя. Для этого есть import() — функция, которая грузит модуль по требованию и возвращает промис.

import() возвращает промис

Динамический import() — это функция, а не инструкция. Она возвращает промис, который выполнится объектом со всеми экспортами модуля (экспорт по умолчанию лежит в свойстве default). Результат удобно дождаться через await — как и ответ fetch.

Ленивая загрузка и разбиение бандла

Главное применение — не грузить всё сразу. Тяжёлый модуль (редактор, график, экспорт в PDF) подключают только в тот момент, когда он реально понадобился. Сборщик видит import() и автоматически выносит такой модуль в отдельный файл (чанк) — стартовая загрузка страницы остаётся быстрой.

Так устроено и это приложение: каждая страница-раздел подключается через import() и грузится лениво — при переходе на неё, а не сразу.

Условный импорт

Поскольку import() — обычный вызов, его можно поставить внутрь if или собрать путь на лету. Так грузят полифилл только старым браузерам или подтягивают нужный файл переводов по языку.

Top-level await

В модулях await разрешён на верхнем уровне — без оборачивания в async-функцию. Это удобно для инициализации (загрузить конфиг, дождаться соединения), но у медали есть обратная сторона: пока такой await не завершится, модули, которые зависят от этого, будут ждать.

Кратко

  • import() — функция, возвращает промис с объектом экспортов (default в свойстве default).
  • Ленивая загрузка: грузим тяжёлый модуль по требованию; сборщик выносит его в отдельный чанк.
  • Условный импорт: import() внутри if или с динамически собранным путём.
  • Top-level await — await на верхнем уровне модуля; задерживает зависящие модули.
