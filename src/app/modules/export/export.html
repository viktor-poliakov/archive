Экспорт

Ключевое слово export помечает то, что модуль отдаёт наружу. Всё остальное остаётся приватным. Экспорт бывает двух видов: именованный (их может быть сколько угодно) и по умолчанию (default, ровно один на модуль).

Именованный экспорт

Самый частый вид. Помечать можно прямо при объявлении (export const, export function, export class) или собрать всё списком в конце файла — тогда сразу видно, что модуль отдаёт. В списке экспорт можно переименовать через as.

Экспорт по умолчанию

export default помечает главную сущность модуля — и он может быть только один на файл. Это удобно, когда файл «про одну вещь»: один класс, одна функция, один компонент. Имя при default не обязательно — важно само значение.

Named или default? Именованные экспорты предпочтительнее: имена фиксированы, их проще искать по коду и они лучше поддаются tree-shaking. default берут, когда в файле действительно одна главная сущность.

default и именованные вместе

В одном модуле можно иметь и default, и именованные экспорты — например, главную функцию по умолчанию плюс вспомогательные рядом.

Ре-экспорт (barrel)

Модуль может пробрасывать чужие экспорты дальше — это ре-экспорт. На нём строят «бочки» (barrel-файлы, обычно index.js): один файл собирает экспорты целой папки, и снаружи всё импортируется из единой точки входа.

Кратко

  • Именованный экспорт — сколько угодно; при объявлении или списком export { … }, с переименованием as.
  • export default — один на модуль, для главной сущности файла.
  • Совмещать default и именованные — можно.
  • Ре-экспорт export … from — собирает экспорты в barrel-файл (index.js).
  • По возможности предпочитайте именованные экспорты.
