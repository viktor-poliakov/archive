Импорт

Ключевое слово import подключает то, что экспортировал другой модуль. Форма записи зависит от того, что именно вы забираете: именованные экспорты, экспорт по умолчанию, всё пространство имён — или вообще ничего, только ради побочного эффекта.

Именованный импорт

Именованные экспорты забирают в фигурных скобках, перечисляя нужные имена. Имена должны совпадать с теми, что указаны в export — но локально их можно переименовать через as (например, если имя занято).

Импорт по умолчанию

Экспорт по умолчанию импортируют без фигурных скобок, и имя вы выбираете сами — оно не обязано совпадать с исходным. Отсюда и гибкость, и риск: один и тот же default в разных файлах могут назвать по-разному.

default и именованные вместе

Если модуль отдаёт и default, и именованные экспорты, их забирают одним import: сначала имя для default, затем список в скобках.

Импорт пространства имён

import * as имя подключает все именованные экспорты как один объект. Обращаться к ним нужно через точку (math.square), зато сразу видно, откуда что взято — это делает код нагляднее, когда импортов много.

Импорт ради побочного эффекта

Иногда модуль ничего не отдаёт — он просто должен выполниться (подключить стили, зарегистрировать что-то, запустить инициализацию). Тогда пишут import без привязки имён.

О путях: путь к модулю относительный (./, ../), и в браузере расширение обязательно ('./math.js', не './math'). «Голые» имена (import x from 'lodash') разрешает сборщик или карта импортов (import maps).

Кратко

  • Именованный: import { a, b } from '…'; имена как в экспорте, переименование через as.
  • default: без скобок, имя выбираете сами.
  • Вместе: import def, { named } from '…'.
  • Пространство имён: import * as ns from '…' → ns.member.
  • Побочный эффект: import '…' — просто выполнить модуль. Путь относительный, расширение обязательно.
