Map и Set

Объекта и массива хватает не всегда. Когда нужен словарь с ключами любого типа — берут Map. Когда нужен набор уникальных значений — Set. Обе коллекции удобнее и предсказуемее, чем «самодельные» решения на объектах и массивах.

Правило простое: нужны нестроковые ключи или частое добавление/удаление — Map; простой конфиг со строковыми ключами — хватит объекта.

Как сравниваются ключи и значения

Map и Set сравнивают по алгоритму SameValueZero — это почти строгое ===, но с одним отличием: NaN считается равным NaN (поэтому им можно пользоваться как ключом/значением). Объекты же сравниваются по ссылке: два разных объекта с одинаковым содержимым — это разные ключи.

Дальше: у Map и Set есть «слабые» версии — WeakMap и WeakSet: они держат объекты, не мешая сборщику мусора их удалить.
