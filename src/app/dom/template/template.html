<template>

<template> — это «заготовка» разметки: кусок HTML, который браузер разбирает, но не показывает на странице. Он лежит наготове, а вы клонируете его столько раз, сколько нужно, — например, под каждый элемент списка.

Аналогия — формочка для печенья. Сама формочка — не печенье, её не едят. Но по ней штампуют одинаковые печенья сколько угодно. <template> — такая формочка для разметки: один образец, много готовых копий.

Что такое <template>

Внутрь <template> кладут любую разметку. Браузер её проверяет и запоминает, но держит «выключенной»: она не видна, картинки не грузятся, скрипты не выполняются, в раскладку страницы она не входит. Это просто образец, ждущий, когда его используют:

Чем это лучше строки с innerHTML

Тот же список можно собрать, склеивая HTML из строк и присваивая innerHTML. Но у шаблона есть заметные плюсы:

  Строка + innerHTML <template>
Разметка склеиваем из строк пишем как обычный HTML
Данные подставляем в строку (риск XSS) через textContent (безопасно)
Разбор HTML заново при каждой вставке один раз, заранее

Главное — безопасность: данные попадают в разметку через textContent, а не подстановкой в строку, поэтому XSS здесь не грозит. Плюс разметку браузер разбирает один раз, а не заново на каждый элемент.

Кратко

  • <template> — разметка-заготовка: разобрана, но не показана и не активна.
  • Содержимое — в .content (это DocumentFragment), а не в обычных детях элемента.
  • Как применять: content.cloneNode(true) → заполнить textContent → вставить в страницу.
  • Безопаснее строк: данные идут через textContent, без риска XSS.
