Массивы: основы

Массив — это упорядоченный список значений, который хранится под одним именем. Если переменная держит одно значение, то массив держит сразу много: список покупок, очередь задач, строки таблицы — всё это удобно описывать массивами.

На этой странице разберём базу: что такое массив, как его создать, как обращаться к элементам и что на самом деле означает свойство length.

Что такое массив

Массив — это упорядоченный список. «Упорядоченный» значит, что у каждого элемента есть своё фиксированное место — индекс. Нумерация идёт с нуля: первый элемент имеет индекс 0, второй — 1, и так далее. Это классическая точка спотыкания для новичков, поэтому запомним сразу: индекс последнего элемента всегда на единицу меньше, чем количество элементов.

Массив может хранить значения любых типов — числа, строки, логические значения, объекты, другие массивы — причём вперемешку в одном массиве. JavaScript это не запрещает.

Аналогия: массив — это пронумерованные ячейки камеры хранения, идущие подряд. Номер ячейки — это индекс, а её содержимое — значение. Зная номер, вы мгновенно достаёте нужную вещь.

Массив — это особый объект

Технически массив в JavaScript — это специальный вид объекта. У этого есть практическое следствие: оператор typeof для массива возвращает 'object', а не что-то вроде 'array'. То есть по typeof отличить массив от обычного объекта нельзя.

Поэтому для проверки «это массив или нет» используют отдельный метод Array.isArray(value). Он принимает любое значение и возвращает boolean: true, если это массив, и false в остальных случаях.

Важно: поскольку массив — это объект, он хранится и копируется по ссылке, а не по значению. Это влияет на то, как массивы передаются в функции и сравниваются. Подробно о ссылках — в разделе Копирование и ссылки.

Как создать массив

Литерал — основной способ

В 99% случаев массив создают с помощью квадратных скобок — это называется литерал массива. Пустой массив — [], массив с элементами — перечисление значений через запятую.

new Array — и его ловушка

Существует и конструктор new Array(...), но у него есть коварная особенность. Если передать одно число, оно понимается не как элемент, а как длина: new Array(3) создаёт пустой массив длины 3, а вовсе не [3]. Если же передать несколько аргументов, они станут элементами, как в литерале.

Именно из-за этой неоднозначности конструктор new Array в обычном коде почти не используют — литерал понятнее и безопаснее. Если нужно создать массив из одного аргумента-числа без ловушки, есть метод Array.of(...): он всегда кладёт переданные аргументы как элементы и возвращает новый массив. Array.of(7) вернёт [7].

Array.from — массив из чего-то другого

Метод Array.from(источник, mapFn?) создаёт новый массив из перебираемого (строка, Set, Map) или похожего на массив объекта. Необязательный второй аргумент — функция преобразования: её применят к каждому элементу на лету. Этим удобно пользоваться, чтобы быстро создать последовательность: объект { length: 5 } — это «похожий на массив» (у него есть только свойство length), а функция (v, i) => i заполняет элементы по индексу — получится [0, 1, 2, 3, 4].

Доступ к элементам

К элементу обращаются по индексу в квадратных скобках: arr[0] — первый, arr[1] — второй. Чтобы достать последний элемент, классически пишут arr[arr.length - 1] — берут индекс «длина минус один».

Современная альтернатива — метод at(). Он принимает индекс и возвращает элемент, но при этом понимает отрицательные индексы, отсчитывая их с конца: arr.at(-1) — последний элемент, arr.at(-2) — предпоследний. Это короче и читается лучше.

Важно: обращение к несуществующему индексу не вызывает ошибку — вы просто получаете undefined. Это удобно, но иногда маскирует опечатки в индексах.

Свойство length

У каждого массива есть свойство length. Его легко принять за «количество элементов», но точнее сказать так: length — это максимальный индекс плюс один. Для плотного массива без пропусков это и есть количество элементов, но как только появляются «дыры», разница становится заметной.

Свойство length можно не только читать, но и записывать. Запись меньшего значения усекает массив (лишние элементы теряются), запись большего — расширяет его пустыми ячейками.

Осторожно: усечение через length необратимо удаляет данные, а расширение создаёт «пустые» ячейки, которые ведут себя не как обычные undefined. Тонкости и подводные камни разобраны в разделе Нюансы и подводные камни.

Изменение и добавление по индексу

Чтобы изменить элемент, достаточно присвоить новое значение по его индексу: arr[i] = value. А вот если присвоить значение по индексу за пределами текущей длины, массив растянется, и между старыми и новым элементами появятся пустые ячейки — те самые «дыры». Свойство length при этом подстроится под новый максимальный индекс.

Добавлять элементы вручную по индексу arr[arr.length] можно, но обычно для этого используют специальные методы вроде push и unshift — они удобнее и понятнее. Им посвящён раздел Добавление и удаление. Подробности про «дыры» (разреженные массивы) — в разделе Нюансы и подводные камни.

Многомерные массивы

Отдельного «двумерного» типа в JavaScript нет — многомерный массив получают просто как массив массивов. Так удобно представлять таблицы, матрицы или сетки. Доступ к элементу идёт в две скобки: arr[i][j] — сначала выбираем внутренний массив по индексу i, затем элемент внутри него по индексу j.

Кратко

  • Массив — упорядоченный список значений; индексы начинаются с 0, элементы могут быть любых типов.
  • typeof arr возвращает 'object'; для проверки используйте Array.isArray(value) (возвращает boolean).
  • Создание: чаще всего литерал []; помните про ловушку new Array(3) (длина, а не элемент); Array.of(...) и Array.from(...) для особых случаев.
  • Доступ: arr[i], последний — arr[arr.length - 1] или arr.at(-1).
  • length — это «максимальный индекс + 1»; запись в него усекает или расширяет массив.
  • Многомерные массивы — это массивы массивов, доступ через arr[i][j].
