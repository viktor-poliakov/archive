Перебор массива

Перебор — это проход по всем элементам массива, чтобы что-то с ними сделать: вывести, посчитать, преобразовать или отобрать нужные. В JavaScript для этого есть и обычные циклы, и удобные методы массива.

Многие из этих методов (forEach, map, filter, reduce) принимают в качестве аргумента функцию — её называют колбэком. Метод сам проходит по элементам и на каждом вызывает вашу функцию. Это и есть функции как значения (функции высшего порядка) на практике.

Циклы

Самый базовый способ обойти массив — обычный цикл. Он не использует колбэков и полностью под вашим контролем.

Классический for

Вы заводите счётчик-индекс i, идёте от 0 до arr.length и обращаетесь к элементам по индексу. Подходит, когда нужен именно индекс или нестандартный шаг (например, через один элемент).

for...of — перебор значений

Цикл for...of перебирает сами значения элементов, без возни с индексом. Это самый удобный способ для простого обхода. Его легко прервать через break или пропустить шаг через continue.

Важно: для массивов не используйте for...in. Он предназначен для перебора ключей объекта: по массиву он пройдёт строковые индексы ('0', '1', …), а заодно прихватит любые лишние свойства, если их кто-то добавил массиву.

forEach — вызвать функцию для каждого элемента

Метод forEach(cb) просто вызывает ваш колбэк для каждого элемента по порядку. Это аккуратная замена циклу, когда нужно «сделать что-то с каждым» (вывести, отправить, записать).

Колбэк получает три аргумента: (element, index, array) — текущий элемент, его индекс и сам массив. Чаще всего пользуются первыми двумя.

  • Что возвращает: undefined — forEach не собирает результат, его вызывают ради побочного действия.
  • Мутация: сам массив не меняет.

Важно: forEach нельзя прервать. Ни break (это синтаксическая ошибка внутри колбэка), ни return не останавливают перебор — return лишь завершает текущий вызов колбэка, и перебор идёт дальше. Если нужно прервать обход досрочно — берите for...of с break либо методы some/every (см. Поиск).

map — преобразовать каждый элемент

Метод map(cb) создаёт новый массив той же длины, где каждый элемент — это то, что вернул колбэк для соответствующего элемента исходного массива. Колбэк получает всё ту же тройку (element, index, array).

  • Что возвращает: новый массив той же длины.
  • Мутация: исходный массив не меняет.

Сравните: forEach ничего не возвращает и нужен ради действия, а map возвращает новый массив и нужен ради преобразования. Если вы используете forEach, чтобы вручную складывать результаты в новый массив, — почти всегда лучше взять map. Подробнее о превращении массива в другие данные — в разделе Преобразование.

filter — отобрать подходящие элементы

Метод filter(cb) создаёт новый массив только из тех элементов, для которых колбэк вернул истинное значение. Колбэк-предикат получает (element, index, array) и должен вернуть true или false (по сути — «оставить элемент?»).

  • Что возвращает: новый массив с отобранными элементами (может быть короче исходного или пустым).
  • Мутация: исходный массив не меняет.

reduce — свернуть массив в одно значение

Метод reduce(cb, initialValue) «сворачивает» весь массив в одно значение: сумму, объект, строку — что угодно. Он накапливает результат, передавая его от шага к шагу.

Колбэк получает (accumulator, element, index, array), где accumulator (аккумулятор) — это накопленный результат с предыдущего шага. То, что колбэк вернёт, станет аккумулятором для следующего элемента.

  • Что возвращает: итоговое накопленное значение.
  • Мутация: исходный массив не меняет.
  • initialValue: стартовое значение аккумулятора (второй аргумент).

Важно: вызов reduce без initialValue на пустом массиве бросает TypeError — брать первый элемент не из чего. Поэтому почти всегда указывайте initialValue: с ним пустой массив безопасно вернёт само начальное значение.

reduceRight

Метод reduceRight(cb, initialValue) работает точно так же, но идёт справа налево — от последнего элемента к первому. Это важно, когда порядок свёртки влияет на результат.

keys, values, entries

Эти три метода возвращают перебираемые объекты (итераторы), по которым удобно ходить через for...of или разворачивать в массив через ... (как устроен сам протокол итерации — в разделе Итераторы и генераторы):

  • keys() — даёт индексы массива (0, 1, 2, …).
  • values() — даёт значения элементов.
  • entries() — даёт пары [индекс, значение].

Особенно полезен entries(): в связке с for...of и деструктуризацией он даёт сразу и индекс, и значение — без ручного счётчика.
