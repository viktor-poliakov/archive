target и currentTarget

Раз событие проходит через много вложенных элементов, полезно различать два вопроса: где событие произошло и кто его сейчас обрабатывает. На эти вопросы отвечают два разных свойства объекта event — и путаница между ними одна из самых частых.

Два разных элемента

  • event.target — элемент, на котором событие реально произошло (самый глубокий, куда вы кликнули). По пути события он не меняется.
  • event.currentTarget — элемент, чей обработчик выполняется прямо сейчас. По мере всплытия он меняется от элемента к элементу.

currentTarget меняется, target — нет

Разница особенно видна, если повесить обработчик на несколько уровней сразу. Пока событие всплывает, target остаётся одним и тем же, а currentTarget на каждом шаге — это текущий элемент:

Отсюда практическое правило: target отвечает на вопрос «что нажали», а currentTarget — «кто обрабатывает». На различии target и currentTarget держится делегирование.

this в обработчике

В обычной функции-обработчике this равен event.currentTarget — тому элементу, на котором висит обработчик:

А вот у стрелочной функции нет своего this — она берёт его из окружающего кода, поэтому на элемент он указывать не будет. Это общее правило стрелочных функций (подробнее — в разделе «Стрелочные функции»). Вывод простой: внутри обработчиков надёжнее опираться на event.currentTarget, а не на this.

Дальше: зная, что target указывает на реальный элемент клика, можно повесить один обработчик на контейнер и разбирать клики по всем детям — это делегирование событий.
