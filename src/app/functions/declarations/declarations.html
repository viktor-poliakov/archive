Объявление функций

Функцию можно создать тремя способами: объявление функции, функциональное выражение и стрелочная функция. Они похожи, но различаются поднятием (hoisting), поведением this и краткостью.

Способы объявления

Объявление функции (function declaration). Поднимается целиком, поэтому её можно вызвать даже выше по коду (см. раздел Hoisting).

Функциональное выражение — функция, записанная в переменную. Доступна только после строки с присваиванием.

Стрелочная функция — короткий синтаксис. Если тело — это одно выражение, его значение возвращается автоматически (без return). Важное отличие: у стрелочной нет своего this (см. Контекст this).

Свой this да да нет (берёт извне)
Можно как метод объекта да да обычно нет

Function Declaration и Function Expression подробнее

Это два разных способа создать функцию, и различает их не вид самой функции, а место, где она написана.

Function Declaration (объявление функции) — это отдельная инструкция, которая начинается с ключевого слова function и имени. Она ничему не присваивается, а просто «объявляет» функцию в текущей области.

Function Expression (функциональное выражение) — это функция, использованная как значение: справа от =, как аргумент другой функции, как элемент массива и т.п. Часто она анонимная (без имени).

Подсказка для распознавания: если строка начинается со слова function — это declaration. Если function стоит где-то внутри выражения (после =, в скобках вызова) — это expression.

Главное отличие — момент доступности

Declaration поднимается целиком, поэтому функцию можно вызвать ещё до строки, где она написана. Expression же ведёт себя как обычная переменная: до присваивания её значение ещё не готово (см. раздел Hoisting).

В примере declared() работает, потому что объявление функции поднялось вместе с телом. А expressed на момент вызова ещё undefined (поднялось только имя переменной var), поэтому попытка его вызвать — это TypeError.

Именованное функциональное выражение

Выражению можно дать имя. Оно будет видно только внутри самой функции — это удобно для рекурсии и понятных названий в стеке вызовов:

Свойство Declaration Expression
Где стоит отдельная инструкция часть выражения (значение)
Поднимается целиком да нет
Вызов до объявления можно нельзя (TypeError / ReferenceError)
Имя обязательно необязательно (часто анонимная)
Типичное применение самостоятельные функции модуля колбэки, функции «на месте», присваивание

Что выбрать: для самостоятельных функций удобнее declaration — их видно по всему файлу. Для функций «по месту» (колбэки, передача аргументом) естественнее expression, в том числе стрелочная.
