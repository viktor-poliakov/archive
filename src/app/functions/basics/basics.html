Функции: основы

Функция — это блок кода с именем, который можно описать один раз и затем вызывать сколько угодно раз. Она принимает входные данные (аргументы), выполняет действия и может вернуть результат. Функции — основной способ разбивать программу на понятные, переиспользуемые части.

Аналогия: функция как рецепт в кулинарной книге. У рецепта есть название («приготовить омлет»), список ингредиентов (параметры) и чёткая последовательность шагов (тело функции), которая приводит к блюду (результату). Записав рецепт один раз, вы готовите по нему сколько угодно раз и с разными ингредиентами. И, как повар, вы знаете каждый шаг внутри — рецепт не прячет приготовление, а описывает его.

Простой пример

Разберём этот пример по частям.

  • Имя функции — add. По нему мы потом и обращаемся к функции. Имя стоит делать осмысленным: оно подсказывает, что функция делает.
  • Параметры — a и b в скобках после имени. Это «заготовки» под входные данные: имена, под которыми функция будет обращаться к тому, что ей передадут. Сами по себе они пустые — значения появляются только при вызове.
  • Тело функции — код в фигурных скобках { … }. Это инструкции, которые выполняются при каждом вызове. Здесь тело состоит из одной строки: return a + b.
  • return и результат — return отдаёт значение наружу и завершает функцию. В примере наружу возвращается сумма a + b. Если бы return не было, функция вернула бы undefined.
  • Вызов функции — add(2, 3). Записать функцию мало: чтобы она выполнилась, её нужно вызвать — указать имя и круглые скобки. Без скобок (add) — это просто ссылка на функцию, а не её запуск.
  • Аргументы — 2 и 3 в скобках при вызове. Это реальные значения, которые подставляются вместо параметров: a станет 2, b станет 3.

Разница между параметрами и аргументами проще всего видна на аналогии с бланком: параметры — это пустые поля «Имя» и «Фамилия» в форме, а аргументы — то, что вы реально вписали в эти поля при заполнении.

Зачем нужны функции

  • Переиспользование: описали логику один раз — вызываете в разных местах, не копируя код.
  • Читаемость: имя функции объясняет намерение. calculateTotal() понятнее, чем десять строк вычислений подряд.
  • Изоляция: у функции своя область видимости, поэтому её внутренние переменные не мешают остальной программе.
  • Абстракция: вызывающему важно «что делает функция», а не «как именно» — детали скрыты внутри.
