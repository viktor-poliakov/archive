JSON на практике

Методы JSON решают несколько очень частых бытовых задач: скопировать объект целиком и сохранить его между перезагрузками страницы (обе — связка stringify + parse), а также наглядно вывести в консоль (тут хватает одного stringify). Разберём каждую — и честно обозначим пределы.

Глубокое копирование

Обычное копирование объекта (присваивание, спред {...obj}) затрагивает только верхний уровень: вложенные объекты остаются общими по ссылке (см. Объекты: копирование и ссылки). Приём «упаковать и распаковать» создаёт полностью независимую копию со всей вложенностью:

…и его ограничения

Копия проходит через JSON, а значит теряет всё, чего в JSON нет: функции, undefined, а Date превращается в строку. Поэтому приём годится только для «чистых данных» — объектов из строк, чисел, булевых, массивов и вложенных объектов:

structuredClone — правильный клон

Для честного глубокого копирования в браузерах и Node есть встроенная функция structuredClone. Она сохраняет Date, Map, Set и даже циклические ссылки — то, на чём приём с JSON спотыкается:

  JSON.parse(JSON.stringify) structuredClone
Вложенные объекты копирует копирует
Date станет строкой сохраняет
Map / Set теряются сохраняет
Функции отбрасываются ошибка
Циклические ссылки ошибка поддерживает

Правило: нужен клон «данных ради данных» и код должен работать в любой среде — годится приём с JSON; нужен точный клон с датами, Map/Set и циклами — берите structuredClone. Функции не копирует ни то, ни другое.
