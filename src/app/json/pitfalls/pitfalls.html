JSON: нюансы и подводные камни

JSON выглядит простым, но у него узкий набор типов, и всё, что в него не влезает, ведёт себя неожиданно: что-то тихо меняет тип, что-то пропадает, а что-то и вовсе роняет программу. Соберём самое частое.

Кратко

  • Даты → строки. Обратно в Date — только вручную через reviver.
  • undefined, функции, символы отбрасываются — в объекте исчезают, в массиве становятся null.
  • NaN и Infinity → null, а BigInt → TypeError.
  • Большие числа теряют точность — длинные ID храните и передавайте строкой.
  • Циклические ссылки → TypeError; для копирования берите structuredClone.
  • Невалидный JSON → SyntaxError (одинарные кавычки, висячая запятая, ключи без кавычек); разбор данных «извне» — в try / catch.
