JSON — основы

JSON (JavaScript Object Notation) — это текстовый формат для хранения и передачи данных. Он выглядит почти как объект JavaScript, но это не объект, а обычная строка, которую понимает любой язык и любая программа. Именно в JSON браузер общается с сервером, а приложения сохраняют настройки.

Аналогия — почтовая посылка. Объект в памяти программы — как вещь у вас дома: пользоваться удобно, но по почте не отправишь. Чтобы переслать её, вещь упаковывают в стандартную коробку (JSON-строку), которую примет любая почта в мире. На другом конце коробку распаковывают обратно в вещь. JSON — это и есть такая универсальная упаковка для данных.

JSON — это просто текст

Главное, что нужно уложить в голове: JSON — всегда строка. Объект и JSON — два представления одних и тех же данных: одно удобно для работы в коде, другое — для пересылки и хранения. Превращение туда-обратно делают два метода: JSON.stringify (упаковать) и JSON.parse (распаковать).

В JSON значением может быть только одно из шести:

  • строка — обязательно в двойных кавычках: "Anna";
  • число — 30, 3.14, -1;
  • булево — true или false;
  • null — «пусто»;
  • массив — список в квадратных скобках: ["cat", "dog"];
  • объект — пары «ключ-значение», ключ — всегда строка в двойных кавычках.

Всё остальное (функции, undefined, даты, Map) в JSON не существует. Как ведут себя такие значения при упаковке — разбираем в JSON.stringify и подводных камнях.

JSON — это не объект JavaScript

JSON похож на объектный литерал, и новички их путают. Но JSON — строгий формат: в нём запрещено многое из того, что разрешено в коде. Вот объект JS, который придётся «причесать», чтобы получить валидный JSON:

  Объект JS JSON
Ключи можно без кавычек: name: только в двойных: "name":
Строки '…', "…", `…` только двойные "…"
Функции / методы да нет
undefined да нет (есть только null)
Комментарии да нет
Висячая запятая да нет

Поэтому JSON нельзя «просто вставить как объект» — его нужно разобрать методом JSON.parse. А чтобы из объекта получить корректный JSON, есть JSON.stringify — он сам следит за всеми этими правилами.
