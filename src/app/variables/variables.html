JavaScript / Переменные

Переменные

Переменная — это именованная ссылка на значение в памяти. Она позволяет сохранить данные и обращаться к ним по имени, а не по конкретному значению. В JavaScript переменные объявляют тремя способами: let, const и устаревшим var.

Объявление

При объявлении переменной задают её имя, а затем (необязательно) присваивают значение через =.

let

Объявляет переменную с блочной областью видимости (живёт внутри {}). Значение можно переприсваивать.

const

Тоже блочная область видимости, но переменную нельзя переприсвоить после инициализации. Значение обязательно задаётся сразу.

Важно: const запрещает переприсваивание ссылки, но не «замораживает» объект. Свойства объекта или элементы массива менять можно.

var и почему его избегают

var имеет функциональную область видимости и поднимается (hoisting) в начало области со значением undefined. Это часто приводит к неочевидным ошибкам.

Затенение (shadowing)

Затенение — это когда переменная во внутренней области видимости имеет то же имя, что и переменная снаружи. Внутренняя переменная «перекрывает» внешнюю: пока мы внутри, видим именно её, а внешняя остаётся нетронутой.

Аналогия: у вас дома есть «настольная лампа», и в гостях у друга тоже есть «настольная лампа». Пока вы у друга, фраза «включи настольную лампу» относится к его лампе. Ваша домашняя лампа при этом никуда не делась — просто сейчас речь не о ней.

Затенение не зависит от того, как объявлена внешняя переменная: будь снаружи let, const или var — новая переменная с тем же именем внутри функции всё равно её перекроет, а внешняя останется нетронутой. Важно лишь то, что внутри объявлена новая переменная.

С var есть важный нюанс: он не различает блоки (if, for и другие) — только функции. Поэтому объявление var внутри блока не создаёт новую переменную, а перезаписывает внешнюю с тем же именем. Это частый источник багов. У let и const такого нет: внутри блока создаётся отдельная переменная.

Вывод: настоящее затенение (когда внешняя переменная сохраняется) var даёт только на границе функции. Внутри обычного блока var вместо затенения молча портит внешнюю переменную.

Переменные и объект window

В браузере есть глобальный объект window — он представляет вкладку и хранит всё «глобальное». Исторически переменные, объявленные через var в глобальной области, автоматически становились свойствами window. С let и const этого не происходит: они остаются в отдельной области и в window не попадают.

Почему это хорошо: window — общий на всю страницу. Если каждая переменная попадает туда, легко случайно перезаписать чужое значение (своё и чужой библиотеки). let и const не «засоряют» глобальное пространство имён, поэтому конфликтов меньше — ещё одна причина предпочитать их вместо var.

Важно: это касается именно глобальной области. На верхнем уровне ES-модуля (<script type="module">) даже var не добавляется в window — там своя область видимости модуля.

Сравнение

Свойство let const var
Область видимости блочная блочная функциональная
Переприсваивание да нет да
Повторное объявление нет нет да
Hoisting да, но без доступа до объявления да, но без доступа до объявления да, со значением undefined

Что использовать

  • const — по умолчанию для всех значений, которые не меняются.
  • let — когда значение действительно нужно переприсваивать (счётчики, накопители).
  • var — не использовать в новом коде.

Правила именования

  • Имя состоит из букв, цифр, $ и _; не может начинаться с цифры.
  • Регистр имеет значение: age и Age — разные переменные.
  • Принят стиль camelCase: userName, totalPrice.
  • Имя должно быть осмысленным: count лучше, чем c.
  • Нельзя использовать зарезервированные слова (let, for, return…).
